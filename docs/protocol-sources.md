# Protocol Source Matrix

This matrix is the normative source index for implementation work. Project specifications explain design intent; official Erlang/OTP documentation defines wire compatibility.

## Initial implementation subset

### ETF

- version byte `131`;
- `SMALL_INTEGER_EXT`, `INTEGER_EXT`;
- `SMALL_ATOM_UTF8_EXT`, `ATOM_UTF8_EXT`;
- `SMALL_TUPLE_EXT`, `LARGE_TUPLE_EXT`;
- `NIL_EXT`, `LIST_EXT`, `BINARY_EXT`;
- `NEW_PID_EXT`.

### EPMD

- `ALIVE2_REQ` / `ALIVE2_RESP` and `ALIVE2_X_RESP`;
- `PORT_PLEASE2_REQ` / `PORT2_RESP`.

### Handshake

- OTP 23+ name/challenge format;
- status exchange;
- MD5 cookie challenge digest as required by the distribution handshake;
- explicit initiating and accepting state machines.

### Connected distribution

- four-byte packet framing and tick packets;
- pass-through distribution header;
- `REG_SEND` and `SEND` control operations for the first echo path.

## Fixture policy

`fixtures/etf/manifest.tsv` is generated from `:erlang.term_to_binary/1`. The manifest records the generator OTP version, source expression, and lowercase hexadecimal bytes. A fixture generated by OTP 28 may validate stable ETF tags, but it does not establish OTP 25–27 interoperability; target-version black-box tests remain required.
