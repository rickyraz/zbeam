# Architecture Decision Records

ADRs record accepted decisions that constrain module boundaries, protocol contracts, ownership, or lifecycle behavior.

## Format

Use `NNNN-short-title.md` with:

- **Status:** proposed, accepted, superseded, or rejected;
- **Context:** observed constraint or conflict;
- **Decision:** normative choice;
- **Consequences:** benefits, costs, and exclusions;
- **Evidence:** tests, measurements, or primary sources;
- **Supersedes/Superseded by:** links when applicable.

Accepted ADRs are immutable except for status and supersession links. New evidence that changes a decision requires a new ADR.
